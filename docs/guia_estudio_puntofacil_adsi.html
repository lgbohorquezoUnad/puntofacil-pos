Guia de estudio de la aplicacion PuntoFacil POS

Material de apoyo academico para ADSI - SENA Colombia

1. Proposito del documento

Este documento explica de forma clara el funcionamiento de la aplicacion PuntoFacil POS. Fue pensado como una guia de estudio para ayudarte a comprender como se construye un sistema web real, como se organizan sus carpetas, que hace cada archivo importante y que conceptos de programacion de software puedes aprender con este proyecto.

2. Que hace la aplicacion

PuntoFacil POS es un sistema web de punto de venta. Permite iniciar sesion, vender productos, abrir y cerrar caja, administrar productos y usuarios, controlar inventario, registrar movimientos de stock y manejar procesos mas avanzados como ordenes de compra, recepcion de mercancia, conteos fisicos, transferencias y auditoria.

3. Arquitectura general

La aplicacion esta organizada en capas. Este es uno de los aprendizajes mas importantes para ADSI.

  • Frontend: muestra las pantallas al usuario con HTML, CSS y JavaScript.
  • Routes: definen las URLs de la API.
  • Controllers: validan solicitudes, revisan permisos y coordinan la logica.
  • Models: ejecutan consultas SQL sobre MySQL.
  • Base de datos: guarda usuarios, productos, ventas, caja e inventario.

Flujo basico: Usuario -> Pantalla -> API Flask -> Route -> Controller -> Model -> MySQL

4. Tecnologias y programas utilizados

  • Python: backend.
  • JavaScript: logica del frontend.
  • HTML y CSS: estructura y estilos.
  • SQL / MySQL: persistencia de datos.
  • Flask: framework principal del backend.
  • flask-cors: comunicacion entre frontend y backend.
  • flask-mysqldb: conexion a MySQL.
  • flask-bcrypt: cifrado de contrasenas.
  • flask-jwt-extended: autenticacion con tokens JWT.
  • Bootstrap 5: estilos responsivos.
  • Bootstrap Icons: iconos.
  • SweetAlert2: alertas visuales.
  • Render: despliegue del backend.
  • Netlify: despliegue del frontend.
  • VS Code: editor sugerido por la carpeta .vscode.

5. Estructura de carpetas

  • backend/: servidor Flask, rutas, controladores y modelos.
  • frontend/: plantillas HTML, CSS y JavaScript.
  • database/: scripts SQL y migraciones.
  • docs/: documentacion del proyecto.
  • .vscode/: configuracion del editor.
  • venv/: entorno virtual de Python.

6. Explicacion de carpetas y archivos

6.1 backend/

Aqui vive la API. Es la parte que recibe solicitudes del frontend y trabaja con la base de datos.

  • app.py: punto de entrada del backend. Crea Flask, carga configuracion, habilita CORS, inicia JWT, conecta MySQL, registra rutas y sirve las paginas HTML.
  • config.py: guarda configuracion sensible mediante variables de entorno, como host de MySQL y claves secretas.
  • requirements.txt: lista las librerias que necesita el proyecto.
  • generate_password.py: genera hashes bcrypt para contrasenas.
  • fix_admin_hash.py: actualiza el hash del administrador directamente en MySQL.
  • .python-version: archivo auxiliar de version de Python.
  • api_stdout.log / api_stderr.log: archivos de depuracion.

6.2 backend/routes/

  • auth_routes.py: define POST /api/login.
  • product_routes.py: agrupa endpoints de productos, categorias, dashboard, inventario, compras, conteos, transferencias y auditoria.
  • sale_routes.py: define el endpoint para registrar ventas.
  • cash_register_routes.py: define endpoints para abrir, cerrar y consultar caja.
  • user_routes.py: define endpoints CRUD de usuarios.

6.3 backend/controllers/

Los controladores reciben la solicitud HTTP y aplican validaciones y permisos.

  • auth_controller.py: valida correo y contrasena, revisa estado del usuario y genera el JWT.
  • product_controller.py: lista, crea, actualiza y elimina productos; tambien entrega categorias y dashboard.
  • sale_controller.py: valida metodo de pago e items, luego registra la venta.
  • cash_register_controller.py: maneja apertura, cierre y consulta de caja.
  • inventory_controller.py: maneja filtros de inventario, ajustes de stock y movimientos.
  • advanced_inventory_controller.py: maneja compras, recepciones, conteos, conciliaciones, transferencias, auditoria y tablero financiero.
  • user_controller.py: administra usuarios y aplica reglas de rol administrador.

6.4 backend/models/

Los modelos trabajan directamente con SQL y MySQL.

  • user_model.py: buscar, listar, crear, actualizar y eliminar usuarios.
  • product_model.py: consultar categorias y productos, y ejecutar el CRUD de productos.
  • sale_model.py: implementa la transaccion de una venta, descuenta stock y guarda detalle.
  • cash_register_model.py: abre y cierra caja, y calcula diferencia entre dinero esperado y cierre real.
  • inventory_model.py: construye resumenes de inventario, aplica ajustes y registra trazabilidad en inventory_movements.
  • advanced_inventory_model.py: implementa logica empresarial avanzada como compras, lotes, transferencias, conteos y auditoria.

6.5 frontend/

  • index.html: redirige a /login.
  • templates/login.html: pantalla de inicio de sesion.
  • templates/pos.html: pantalla de punto de venta.
  • templates/admin.html: pantalla administrativa.
  • templates/inventory.html: pantalla de control de inventario.
  • templates/operativa.html: pantalla de procesos avanzados.

6.6 frontend/static/js/

  • config.js: calcula la URL base de la API para local o produccion.
  • auth.js: guarda token y usuario en localStorage, valida permisos y adjunta cabecera Authorization.
  • login.js: controla el formulario de inicio de sesion y la redireccion segun rol.
  • pos.js: carga productos, maneja carrito, caja, medios de pago y checkout.
  • admin.js: controla dashboard y CRUD de productos y usuarios.
  • inventory.js: controla filtros, KPIs, movimientos, ajuste de stock y exportacion CSV.
  • operativa.js: controla compras, recepcion, conteos, transferencias y auditoria.
  • ui.js: centraliza toasts, alertas, confirmaciones, tema y modo visual.

6.7 frontend/static/css/

  • app-shell.css: estilos globales compartidos entre las pantallas.

6.8 database/

  • schema.sql: estructura base de cajas, ventas, detalle de venta y movimientos de inventario.
  • migration_users.sql: crea la tabla usuarios y el administrador inicial.
  • migration_user_status.sql: agrega el estado de usuario.
  • migration_add_cash_register.sql: incorpora la tabla cajas y relacion con ventas.
  • migration_inventory_control.sql: agrega stock minimo y movimientos de inventario.
  • migration_high_impact.sql: agrega proveedores, ordenes de compra, lotes, ubicaciones, transferencias, conteos y auditoria.

6.9 Otros archivos de la raiz

  • render.yaml: configura el despliegue del backend en Render.
  • netlify.toml: configura la publicacion del frontend en Netlify.
  • .vscode/settings.json: define el puerto 5503 para Live Server.
  • venv/: entorno virtual, no contiene logica del negocio.

7. Funcionamiento paso a paso

Inicio de sesion

  1. El usuario abre la pantalla login.html.
  2. login.js toma correo y contrasena.
  3. Se envia POST /api/login.
  4. auth_controller.py valida datos y compara la contrasena con bcrypt.
  5. Se genera un token JWT.
  6. auth.js guarda la sesion y redirige segun rol.

Venta

  1. El cajero agrega productos al carrito en pos.html.
  2. pos.js arma el payload con metodo de pago e items.
  3. Se envia POST /api/sales.
  4. sale_controller.py valida la solicitud.
  5. sale_model.py verifica caja abierta y stock disponible.
  6. Se insertan registros en ventas y detalle_venta.
  7. Se actualiza el stock de los productos.

Inventario

  1. El administrador abre inventory.html.
  2. inventory.js consulta KPIs, productos y movimientos.
  3. El backend filtra, resume y devuelve JSON.
  4. Si hay ajuste de stock, se actualiza el producto y se registra el movimiento historico.

Operaciones avanzadas

  1. En operativa.html se crean ordenes de compra.
  2. Se cargan y recepcionan ordenes.
  3. Se abren conteos fisicos y luego se concilian.
  4. Se registran transferencias entre ubicaciones.
  5. Se consultan logs de auditoria para trazabilidad.

8. Que puedes aprender de ADSI con este proyecto

  • Arquitectura cliente servidor.
  • Separacion por capas.
  • Autenticacion y autorizacion.
  • CRUD y validaciones.
  • Consumo de API REST con fetch.
  • JSON y manejo de sesiones.
  • Transacciones y relaciones en MySQL.
  • Control de inventario y trazabilidad.
  • Configuracion por ambientes y despliegue.

9. Ruta recomendada de estudio

  1. Estudia backend/app.py para entender el punto de entrada.
  2. Revisa login completo: login.html, login.js, auth_controller.py y user_model.py.
  3. Estudia la venta completa: pos.js, sale_controller.py y sale_model.py.
  4. Luego revisa CRUD de productos con product_controller.py y product_model.py.
  5. Despues estudia inventario con inventory_model.py.
  6. Finalmente analiza advanced_inventory_model.py para ver procesos empresariales avanzados.

10. Conclusion

PuntoFacil POS es una aplicacion completa y muy util como ejemplo de estudio. Te permite ver como una solicitud entra desde una pantalla, pasa por autenticacion, se valida en controladores, se procesa en modelos, se guarda en MySQL y vuelve al usuario en forma de respuesta.
